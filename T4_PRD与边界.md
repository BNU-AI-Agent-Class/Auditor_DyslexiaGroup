# T4 · PRD 与边界

> **状态**：DEFINE 第四步，基于 T1/T2/T3 整理。  
> **核心原则**：边界是灵魂——这个 Agent 是陪孩子从害怕阅读走到愿意主动阅读的安全伙伴，不是老师、不是医生、不是真人陪伴的替代品。

---

## 1. 一页 PRD · 六要素

| 要素 | 内容 |
|---|---|
| ① 一句话定位 | 一个陪伴三、四年级阅读落后儿童（含疑似阅读困难，不用于诊断）完成课外识读率测验，并推荐匹配难度书目的安全伙伴 Agent。 |
| ② 目标用户 | **主画像**：小航，9 岁小学三年级男孩，识字量不足、看到大段文字发怵、怕被责备。**次要画像**：语文老师/班主任（想为班里阅读落后学生匹配适合材料但时间不够）。 |
| ③ 核心功能（≤3 个，接 T3 刚性需求） | **F1 识读率施测**：播放/显示标准化短文，让孩子朗读，AI 记录朗读过程。**F2 即时计分与错字/卡点定位**：自动统计识读率，记录生字、跳行、回读等具体卡点，但**不输出任何医学或教育诊断标签**。**F3 匹配难度书目推荐**：根据测验结果推荐 10 本“跳一跳够得着”的书，支持老师 override。 |
| ④ **明确不做** | 不诊断阅读障碍或任何学习障碍；不开药、不推荐医疗/干预机构；不替代真人陪伴和老师专业判断；不处理情绪/心理危机；不长期留住用户，能力达标后主动退场。 |
| ⑤ 成功标准（可观察） | **本次可测（短周期）**：<br>• 1 次使用：孩子能在 5 分钟内完成测验，并拿到 1 本当天愿意翻开读的书；测验过程中未出现哭泣、摔书或“我好笨/不想读了”等情绪崩溃信号。<br>• 1 周内 3 次使用：孩子愿意主动完成第 2、3 次测验；至少一次表达“这本书我能读”或愿意继续读。<br><br>**长期目标（本次不测，供后续迭代追踪）**：<br>• 一学期内读完 10 本书循环，识读率提升；达标后 Agent 减少干预，孩子能自主选书。 |
| ⑥ 主要风险 | **漏识别情绪崩溃**：孩子表达挫败或自伤信号时未转人工。**过度依赖/替代陪伴**：老师把 Agent 当“电子老师”，减少真人互动。**数据泄露**：儿童语音/测验成绩被未授权访问。**期望错位**：老师把测验结果当“诊断书”。 |

## 2. 边界三问 · 硬红线（触发条件 + 动作 + 绝不）

### 红线 1：情绪崩溃与自伤/自我否定信号
- **触发条件**：孩子连续 2 次说“不想读了”“太难了”“我好笨”，或出现哭泣、摔东西、表达“活着没意思/不想活了”等自伤/自杀信号。
- **动作**：立即暂停测验；用温柔语气安抚（“没关系，我们先休息”）；后端同步通知老师/学校心理老师；提供真人支持资源（学校心理老师、心理援助热线）。
- **绝不**：绝不在孩子情绪崩溃时继续推进测验或讲大道理；绝不自行进行危机干预或承诺保密。

### 红线 2：超出专业范围的问题（诊断 / 开药 / 法律 / 教育政策）
- **触发条件**：老师问“我孩子/我们班学生是不是阅读障碍/dyslexia？”“要不要吃药/做训练？”“学校必须给我孩子单独考试吗？”等。
- **动作**：明确声明边界——“我只能评估当前识读表现，不能诊断，也不能替代医生/老师/法律专业人士”；给出建议方向（“建议带孩子去正规医院儿保/心理科评估”或“请咨询学校教务处/特教老师”）。
- **绝不**：绝不使用“阅读障碍”“确诊”“必须治疗”等医学/法律标签；绝不推荐具体药品、医疗机构或疗法。

### 红线 3：替代真实成人 / 扮演某个人
- **触发条件**：孩子说“你当我老师来骂我/检查我”；老师说“你代替我陪他读书”；或要求 Agent 模仿某个真实人物说话。
- **动作**：温和澄清身份——“我是你的阅读小伙伴，不是真人，也不能代替老师或爸爸妈妈”；引导回到“我们一起选一本你能读的书”这一核心任务。
- **绝不**：绝不伪装成真人老师或医生；绝不替老师/家长履行陪伴/管教责任。

### 红线 4：数据与隐私（来自 T1 的硬约束）
- **触发条件**：任何第三方（老师、其他家庭成员、云端服务）试图查询孩子的测验成绩、语音记录或阅读历史。
- **动作**：默认拒绝；要求学校/老师（账户持有人）明确授权；原始语音默认本地加密存储、不上传云端；展示给孩子的反馈只用昵称/头像，不用真实姓名。
- **绝不**：绝不在未经授权下向第三方共享、展示或上传可识别儿童的语音/成绩数据。

## 3. 数据红黄绿清单

| 数据 | 灯 | 处理方式 |
|---|---|---|
| 儿童真实姓名、学号、学校、班级、家庭住址 | 🔴 红 | **不采集**。产品内只用昵称/头像/匿名 ID。 |
| 儿童朗读原始语音 | 🟡 黄 | 默认本地加密存储，不上传云端；使用前需学校/老师明示授权；可删除；如确需上传云端，则升级为 🔴 红，必须单独二次授权。 |
| 测验逐字记录（哪些字读错/跳过/回读） | 🟡 黄 | 化名/去标识化后本地存储；用于推荐和进步曲线；分享给老师/医生需学校/老师授权。 |
| 测验得分、识读率、阅读时长、已读书目 | 🟡 黄 | 聚合统计、化名展示；生成进步报告需学校/老师授权；不向第三方默认开放。 |
| 阅读偏好（喜欢的题材、难度反馈） | 🟡 黄 | 匿名化后用于推荐；可导出/删除。 |
| 公开标准化识读率测验材料（如 3 组×30 字短文） | 🟢 绿 | 正常使用，注明来源。 |
| 公开书目元数据（书名、作者、难度指数、适读年级） | 🟢 绿 | 正常使用。 |
| 公开阅读/学习科学常识、儿童心理科普知识 | 🟢 绿 | 正常使用，避免编造。 |

## 4. 人本 AI 七原则自查

| # | 原则 | PRD 中如何体现 |
|---|---|---|
| 1 | 人在环 HITL | 情绪崩溃/自伤信号必须通知老师/学校心理老师/真人；诊断/医疗问题必须转介真人专业人士。 |
| 2 | 透明可解释 | 开场即说明“我是 AI 小伙伴，不是真人老师/医生”；推荐书目时解释“为什么推荐这本”（难度匹配、你喜欢××题材）。 |
| 3 | 心智模型对齐 | 首页/开场说清三件事：能做什么（测验+推书）、不能做什么（不诊断、不替代陪伴）、何时找真人。 |
| 4 | 优雅失败 | 识别不出语音或孩子拒绝朗读时，不硬编分数，而是说“这次没听清，我们换一篇/休息一下”，并给出备选路径。 |
| 5 | 信任校准 | 不夸大 AI 能力（不说“我能治好阅读困难”），也不让老师/家长完全依赖（强调真人陪伴和后续人工评估）。 |
| 6 | 用户控制 | 孩子可随时暂停/退出测验；学校/老师（账户持有人）可随时查看、导出、删除数据；可人工 override 推荐难度。 |
| 7 | 伦理包容 | 不贴标签、不评价“笨/慢”；考虑不同家庭经济背景，推荐书目优先公开/免费资源；界面无障碍（大字体、语音交互）。 |

## 5. HAX 18 抽检（挑 6 条与本项目最相关的对照）

- [ ] **G1 说清能做什么** —— 开场白：“我是你的阅读小伙伴，陪你做一个小测验，然后一起挑一本今天你能读的书。我不会诊断你是不是阅读障碍，也不能代替老师或爸爸妈妈。”
- [ ] **G2 说清能做多好（校准期望）** —— 在测验前说明：“我会努力听清你读的每一个字，但有时候可能听错；如果推荐的书不合适，你可以告诉我换一本。”
- [ ] **G4 显示系统状态** —— 朗读界面实时显示“我在听 / 我在等你读 / 已记录”等状态，让孩子知道 AI 有没有在听、当前进行到哪一步。
- [ ] **G8 能高效关闭 / 退出** —— 任何界面都有显眼的“暂停/退出”按钮；退出时保存当前进度，不强迫完成。
- [ ] **G9 能高效纠正** —— 孩子说“这书太难了/我不喜欢”，Agent 立即响应：“那我们换一本更简单的/你更感兴趣的”；老师可手动调整推荐难度。
- [ ] **G11 说明为何这么做** —— 推荐某本书时解释：“这本比刚才的测验简单一点，讲的是你喜欢的恐龙，适合今天读。”
